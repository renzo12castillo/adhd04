¿Qué es TypeScript?

TypeScript es un lenguaje de programación de código abierto desarrollado por Microsoft que se basa en JavaScript. Su principal característica distintiva es que añade tipado estático opcional y otras características propias de los lenguajes orientados a objetos, como clases, interfaces, y herencia, que no están presentes de forma nativa en JavaScript (al menos no con la misma robustez) TypeScript es un superset (superconjunto) de JavaScript. Esto significa que todo código JavaScript válido también es válido en TypeScript. Sin embargo, TypeScript agrega capacidades adicionales.

La característica más relevante es el tipado estático. Permite declarar tipos explícitamente para variables, parámetros, funciones y objetos. Esto ayuda a detectar errores antes de ejecutar el código, durante el proceso de desarrollo. Gracias al tipado estático y a la inferencia de tipos, los editores de código como VSCode ofrecen autocompletado, refactorizaciones seguras y navegación más sencilla entre archivos.

¿Qué ventajas ofrece TypeScript frente a JavaScript puro?

TypeScript ofrece numerosas ventajas frente a JavaScript puro, especialmente cuando se trata de desarrollar aplicaciones grandes, complejas o de larga duración.

    Tipado estático

  • TypeScript permite declarar tipos para variables, funciones y objetos.
  • Ayuda a detectar errores en tiempo de compilación, antes de ejecutar el código.
  • Mejora la claridad del código al mostrar la intención del desarrollador.

    Detección temprana de errores

  • El compilador de TypeScript encuentra errores comunes durante la compilación.
  • Evita muchos fallos típicos de JavaScript que solo aparecen en tiempo de ejecución.

    Mayor mantenibilidad del código

  • El tipado y las interfaces claras hacen que el código sea más legible, escalable y mantenible
  • Facilita trabajar en equipo y entender código escrito por otros desarrolladores

    Soporte para Programación Orientada a Objetos (POO)

  • TypeScript añade características como clases, interfaces, herencia, modificadores de acceso (public, private, protected) y abstracción.
¿Cómo se instala TypeScript en VSCode?

  1. Descarga Node.js desde https://nodejs.org/ (versión LTS recomendada).
  2. Ejecuta el instalador y sigue las instrucciones.
  3. Verifica que Node.js y npm (Node Package Manager) estén instalados ejecutando en tu terminal: node -v npm -v
  4. Abre una terminal (puedes usar la terminal integrada de VSCode con Ctrl + `` )
  5. Ejecuta el siguiente comando para instalar TypeScript de manera global: npm install -g typescript
  6. Verifica la instalación con: tsc -v
  7. Crea una carpeta para tu proyecto y ábrela en VSCode.
  8. Inicializa un proyecto TypeScript ejecutando en la terminal: tsc --init (Esto creará un archivo tsconfig.json con la configuración predeterminada).
  9. Abre VSCode y ve a Extensiones (Ctrl + Shift + X).
  10. Busca "TypeScript" e instala la extensión oficial de Microsoft.
  11. Crea un archivo .ts (ej. index.ts) y escribe código TypeScript.
  12. Compila el archivo ejecutando en la terminal: tsc index.ts (Esto generará un archivo index.js).
  13. Para compilar automáticamente en modo observación (watch), usa: tsc -w

¿Cómo se compila un archivo TypeScript para convertirlo a JavaScript?
  1. Ejecuta en la terminal el comando tsc seguido del nombre del archivo: tsc mi-archivo.ts Esto generará un archivo mi-archivo.js en la misma carpeta.
  2. Si quieres que TypeScript recompile automáticamente cada vez que guardas cambios, usa: tsc mi-archivo.ts --watch O EN SU LUGAR tsc mi-archivo.ts -w
  3. Si tienes un archivo tsconfig.json (generado con tsc --init), simplemente ejecuta: tsc Esto compilará todos los archivos TypeScript del proyecto según la configuración definida. Para activar el modo observación en todo el proyecto: tsc --watch o tsc -w
  4. Especificar la carpeta de salida: tsc --outDir ./carpeta-salida
  5. Compilar en un módulo específico (ej. ES6): tsc --target ES6
  6. Generar archivos .js con mapas de origen (para depuración): tsc --sourceMap
¿Qué extensión de archivo se utiliza para los archivos TypeScript?
  • Los archivos de TypeScript utilizan la extensión: ts
  • Ejemplo: app.ts, usuario.ts, servicio.ts
¿Cómo se define un tipo de dato en TypeScript?

En TypeScript, los tipos de datos se definen utilizando anotaciones de tipo. Estas anotaciones permiten especificar qué tipo de valor puede contener una variable, parámetro, función, objeto, etc. Esto proporciona seguridad en tiempo de compilación y ayuda a detectar errores antes de ejecutar el código.

  • Tipos primitivos
    •                                     let nombre: string = "Juan";
                                    let edad: number = 30;
                                    let activo: boolean = true;
  • Arreglos (Arrays)
    •                                     let numeros: number[] = [1, 2, 3];
                                    let nombres: string[] = ["Ana", "Luis"];
  • Tuplas
    •                                     let persona: [string, number] = ["Carlos", 40];
  • Enum
    •                                     enum Color { Rojo, Verde, Azul }
                                    let colorFavorito: Color = Color.Verde;
  • Any (tipo flexible, sin verificación)
    •                                     let valor: any = "Hola";
                                    valor = 25; // válido, pero menos seguro
  • Unknown (más seguro que any)
    •                                     let dato: unknown;
                                    dato = "Texto";
                                    dato = true;
  • Funciones (con tipos en parámetros y retorno)
    •                                     function sumar(a: number, b: number): number {
                                        return a + b;
                                    }
  • Objetos
    •                                     let usuario: { nombre: string; edad: number } = {
                                        nombre: "Laura",
                                        edad: 28
                                    };
  • Interfaces
    •                                     interface Persona {
                                        nombre: string;
                                        edad: number;
                                        }
                                    let p1: Persona = { nombre: "José", edad: 35 };
  • Tipos personalizados (type)
    •                                     type ID = string | number;
                                    let userId: ID = 123;
                                    userId = "abc";
  • Uniones de tipos
    •                                     let valor: string | number;
                                    valor = "hola";
                                    valor = 42;
  • Tipos literales
    •                                     let direccion: "norte" | "sur" | "este" | "oeste";
                                    direccion = "norte"; // válido
                                    // direccion = "arriba"; // Invalido
  • Funciones con parámetros opcionales o por defecto
    •                                     function saludar(nombre: string = "Invitado"): void {
                                    console.log("Hola " + nombre);
                                    }
  • Funciones que no retornan (void)
    •                                     function mostrarMensaje(): void {
                                    console.log("Mensaje mostrado");
                                    }
  • Funciones que nunca terminan (never)
    •                                     function error(mensaje: string): never {
                                    throw new Error(mensaje);
                                    }
